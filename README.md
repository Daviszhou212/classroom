# 班级电子宠物管理系统（离线 MVP）

## 运行方式
- 方式一：直接双击打开 `index.html`。
- 方式二：本地启动静态服务器（仍离线），例如：
  - `python -m http.server`
  - 或使用任意本地静态服务器工具

## 功能概览
- 教师模式：学生管理（含加分）、班会喂养模式入口、导入导出、系统设置。
- 学生展示：查看宠物与积分（仅展示）。
- 班会喂养模式：老师开启后，学生轮流自选自己；全班需先完成宠物认领，之后学生可先兑换食物存入个人背包，再从背包里连续喂养或跳过本次操作。
- 展示模式：课堂投影轮播/翻页展示（只读）。
  - 仅从主页进入，退出后返回主页。
  - 现为多卡片分页展示（上一页/下一页）。
- 宠物类型：当前内置 8 种动物 `rabbit / panda / raccoon / capybara / cat / turtle / dog / bird`，界面会在每次进入宠物页面时为同类宠物随机轮换一张 PNG 形象。
- 教师登录支持恢复码重设 PIN：
  - 未登录时可在教师入口点击 `重新设置 PIN`
  - 输入管理员提供的恢复码后，可直接重设新的教师 PIN
  - 如果本地 `teacherPinHash` 配置损坏，系统会提示使用恢复流程修复

## 数据与存储
- 数据保存在本机浏览器 LocalStorage。
- 支持导出/导入 JSON 备份恢复，包含学生食物库存与加分批次/条目撤销记录。
- 教师模式中的反馈建议会先保存在当前浏览器，并随 JSON 备份一起导出/导入。
- 提交反馈时，系统会优先尝试把反馈同步追加到本地 `JSONL` 文件；如果当前浏览器或打开方式不支持持续写入同一文件，则会自动下载一条 `JSONL` 反馈文件作为副本。
- 支持导入学生名单 CSV（格式：`studentNo,name,group,alias`，其中 alias 为拼音/英文名，可选）。示例文件：`data-samples/students.csv`。
- 导入学生名单或手动新增学生后，系统会先创建“待认领”宠物档案；老师需先完成宠物认领，才会开放班会喂养、展示模式、学生查看和学生详情。
- 导入 JSON 时会校验教师 PIN 配置；无效的 `teacherPinHash` 会被拒绝导入。
- 旧数据中的 `hamster` / `fish` 或其他无效 `petType` 会在加载或导入时自动迁移为新的 8 种宠物之一。
- 所有教师加分都会生成一条可审计的加分批次记录；批次内每条加分默认都有固定 7 天撤销窗口。
- 撤销窗口依赖本机浏览器时间计算；纯离线模式下不处理系统时间被手动修改的情况。

## 约束与设定
- 完全离线运行，不依赖外网资源。
- 不启用扣分功能。
- 展示模式只读，不允许喂养或积分变更。
- 喂养只在班会喂养模式中进行，不提供教师代喂入口。
- 教师模式内会在右上角提供“返回仪表盘”和“退出教师模式”两个固定入口；前者位于左侧，后者位于右侧。
- 班会喂养需由老师先开启，并由老师手动结束会话。

## 使用提示
1. 首次进入教师模式需要设置教师 PIN。
2. 添加学生后会自动生成待认领的宠物档案；认领完成前，班会喂养、展示模式、学生查看和学生详情会被锁定。
3. 宠物认领完成后，班会喂养模式才能从教师首页进入；学生在老师监督下轮流选择自己，先兑换食物，再从个人背包中喂养。首次有效喂养前仍可改 1 次宠物。
4. 教师模式中的“近 7 天加分记录”会按批次分组展示，但撤销是按条目执行；同批里某一条不可撤销时，不影响其他条目继续撤销。
5. 展示模式只能从主页进入，退出后返回主页，并保持只读。
6. 教师模式内不提供主页或展示模式出口；如需返回教师仪表盘，使用右上角更醒目的“返回仪表盘”，如需离开教师模式，使用其右侧“退出教师模式”。
7. 如果忘记教师 PIN，或系统提示教师 PIN 配置损坏，可在教师登录页点击 `重新设置 PIN`，使用恢复码重新设置后直接进入教师模式。
8. 教师仪表盘中的 `反馈建议` 只在教师模式内提供，用于记录本机设备上的改进建议，不会联网发送。
9. 教师仪表盘会持续显示数据安全状态；建议在重要操作前先执行一次 `立即备份`，避免浏览器清理站点数据后无法恢复。
10. 如在 Chromium 类浏览器并通过 `http://localhost` 访问，反馈可持续写入同一个 `JSONL` 文件；直接双击 `index.html` 或非 Chromium 环境会自动退化为下载单条反馈文件。

## 素材说明
- 运行时宠物图片使用本地 PNG 多变体资源，路径为 `assets/pets/<type>/`。
- 每种宠物内置 4 张形象图，当前用于页面进入时的随机轮换展示。
